The intuition is easier than the formal statement. Start there.

## The intuition — matching on observed confounders

Determine which other variables $X$ (age, gender) drive both $D$
(a drug) and $Y$ (health). Then, find units with the same values for
$X$ (same age, same gender), but different values for $D$, and compute
the difference in $Y$. If there is a difference in $Y$ between these
units, it should be due to $D$, and not due to anything else.

That's the whole causal-inference programme in one paragraph. Almost
everything below is the formal version of it — with two additions that
the intuition doesn't make obvious:

1. Not every observed variable is safe to match on. Some *actively
   bias* the estimate when you include them.
2. Sometimes no valid matching set exists at all — and the graph is
   what tells you.

The **backdoor criterion** is the concrete recipe for both.

## From intuition to graph

Switching to formal notation for the technical part: call the
treatment $X$ (previously $D$), the outcome $Y$, and the set of
observed variables we might adjust on $Z$ (previously $X$).

We want to estimate the interventional distribution $P(y \mid do(x))$
from observational data. We have a DAG that describes the true
data-generating process. Which subset $Z$ of observed variables gives
us an unbiased estimate?

## The criterion

A set of observed variables $Z$ satisfies the backdoor criterion
relative to the pair $(X, Y)$ if:

1. **No node in $Z$ is a descendant of $X$.** Conditioning on
   consequences of the treatment biases the estimate — this is the
   classic mistake of "controlling for the mechanism".
2. **$Z$ blocks every path from $X$ to $Y$ that begins with an arrow
   *into* $X$.** These are the *backdoor paths* — spurious routes from
   $X$ to $Y$ through common causes.

If such a $Z$ exists, the causal effect is identified:

$$
P(y \mid do(x)) \;=\; \sum_{z} P(y \mid x, z)\, P(z).
$$

This is the **adjustment formula**. Regress $Y$ on both $X$ and $Z$,
then average over the marginal distribution of $Z$. In the matching
intuition above, "find units with the same $X$" is the special case
where you condition on $Z$ discretely rather than modelling it.

## The colliders trap — why "match on everything" is wrong

The intuition above suggests: measure everything you can, match on
all of it, done. But some inclusions actively *hurt* the estimate.

The offenders are called **colliders** — common-effect nodes. A path
is blocked by $Z$ if any node on the path is either:

- a non-collider that *is* in $Z$, or
- a collider that is *not* in $Z$ (and none of its descendants are in
  $Z$ either).

Conditioning on a collider *opens* a path rather than blocking it —
the Berkson-style bias. So a valid adjustment set has to include the
confounders while carefully *not* including the colliders. The graph
is what tells them apart. Without a graph, "throw in every observed
variable" is a strategy that biases you in ways you can't predict.

## When the intuition can't be saved

The backdoor criterion has a hard requirement: all the backdoor paths
must be blockable by observed variables. If some backdoor path runs
entirely through unobserved variables, no backdoor adjustment can save
you — no matter how much you match on what you *can* see.

You need something else — an instrument, the frontdoor criterion,
sensitivity analysis — or you have to admit the effect isn't
identifiable from this data. This is the honest part of causal
inference: it tells you when your data isn't enough. Regression will
produce a number regardless.

## The mental habit

The backdoor criterion is worth internalising as a mental habit even
when you don't draw the graph explicitly: whenever someone claims a
causal effect from observational data, ask *"what are the backdoor
paths, and did they close them all?"* If they can't name the paths,
they didn't think about it. If they closed some but not all, the
estimate is biased in a direction the graph will tell you.
